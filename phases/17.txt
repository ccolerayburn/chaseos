### Intent / art direction

### Hard design rule (do not deviate)

Display 1 is public. The art must contain **no readable text** and **no general local
photos**. The innovation takeaway influences the art **only as a deterministic seed**:
hash the existing public-safe takeaway string to an int and seed `random.Random(seed)`.
Never render the takeaway, the raw check-in, or any private text onto Display 1. Preserve
the existing invariant that Display 1 never sources general Lightroom/local photos.

- `app/readiness.py` (~130, 147â€“155, 198â€“215) and `/daily status` read the same path/meta.

**Therefore: write the art PNG to the same path/filename the poster engine uses

## Workstream B â€” pin-to-taskbar, run on demand (not a startup script)

### Intent

ChaseOS should be launched manually each day from a pinned taskbar icon, not auto-run at
login. Nothing currently auto-runs unless `/startup enable` was invoked, so this is mostly:
ship a windowless, branded, pinnable launcher and set up clean taskbar pinning.

### Changes

1. `pyproject.toml`
   - Add a GUI (no-console) entry point so a windowless `chaseos.exe` is produced in
     `.venv\Scripts\`:
     ```toml
     [project.gui-scripts]
     chaseos = "chaseos.__main__:main"
     ```
   - Keep the existing `[project.scripts]` console entry if useful for headless CLI, or
     rename it (e.g. `chaseos-cli`) to avoid two `chaseos.exe` collisions â€” verify which
     entry table wins on Windows and ensure the pinnable target is the GUI one.

2. `src/chaseos/app/tray_app.py`
   - At the very start of app init (before setting the window/app icon), set an explicit
     AppUserModelID so the taskbar pin is stable and branded:
     ```python
     import ctypes
     try:
         ctypes.windll.shell32.SetCurrentProcessExplicitAppUserModelID("ChaseCole.ChaseOS")
     except Exception:
         pass  # non-Windows / unavailable
     ```
   - Use a real `.ico` for `QApplication.setWindowIcon` and the tray icon (see #3); keep
     `create_fallback_icon()` only as a fallback when the asset is missing.

3. Add `src/chaseos/assets/chaseos.ico` (real multi-resolution icon). Reference it from the
   tray app and the shortcut creation. Ensure it's included in packaging (package-data /
   `MANIFEST.in` / setuptools package data as appropriate).

4. `src/chaseos/windows/startup_shortcut.py`
   - Generalize `StartupShortcutManager` so the destination folder is a parameter:
     Startup folder vs. Start Menu Programs vs. Desktop. Reuse the existing PowerShell /
     WScript COM backend.
   - Add command support for `/install shortcut` and `/uninstall shortcut` (wire through
     `command_router.py` + `headless.py`) that writes a **Start Menu Programs** shortcut
     named `ChaseOS.lnk`, target = the GUI `chaseos.exe`, icon = `chaseos.ico`,
     working dir = project root.
   - Keep `/startup enable|disable|status` available but separate; do not auto-enable it.
   - NOTE: modern Windows has no public API to programmatically pin to the taskbar. The
     correct UX is: command creates the branded Start-Menu shortcut, user right-clicks â†’
     "Pin to taskbar" once. Document this in the README and the command output.

5. Optional (nice-to-have, only if time permits)
   - Second-launch focus: `SingleInstanceGuard` (`windows/single_instance.py`) currently
     just blocks a second instance. Improve so a second click signals the running instance
     to show/focus the terminal window (lightweight local socket or watched flag file).
   - Add a `--start` flag (and/or default first-launch behavior) so clicking the pinned
     icon opens the terminal window and drops the user at the ritual rather than a bare
     tray icon.
