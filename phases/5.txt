Goal:
Replace the placeholder public poster plan with a real local poster engine.

ChaseOS should generate a public-safe 1080x1920 portrait PNG for Display 1 based only on the innovation exercise takeaway.

This poster is intended to be visible to anyone visiting the user's cube.

The poster should:
- be exactly 1080x1920
- be portrait
- be sleek
- be minimal
- be futuristic
- be cyberpunk-inspired
- have high readability from several feet away
- include an original short quote
- include an optional small subtitle
- never include private check-in details
- never include sensitive workplace/private data

Do not implement:
- OpenAI calls
- AI image generation
- private wallpaper generation for Displays 4/2/3
- Lightroom indexing
- Windows wallpaper application
- registry changes
- taskbar icon changes
- clinical questionnaires
- diagnosis language

Important privacy/product rule:
The public poster must be based only on the innovation takeaway.

The raw check-in text must never be used in:
- public poster quote
- public poster subtitle
- public poster metadata intended for public display
- rendered image text
- public-safe takeaway
- public poster prompt/plan

The private innovation takeaway can be stored in metadata locally, but the rendered poster image must contain only public-safe content.

Existing flow:
The ritual currently reaches:
- INNOVATION
- INNOVATION_TAKEAWAY
- POSTER_PLAN
- POSTER_APPROVAL

In Phase 5:

4. Generate a text-described poster plan.
5. Ask for approval/change/regenerate.
6. On approval, render the final 1080x1920 PNG.
7. Store poster metadata locally.
8. Continue the ritual.

Ask:
chaseos> approve poster, change quote, or regenerate?
chaseos> commands: /approve, /change <request>, /regenerate, /poster, /skip

Poster approval behavior:
- /approve renders the poster PNG and moves to WORK_RAMP.
- /skip renders the current poster PNG and moves to WORK_RAMP.
- /poster prints current poster plan.
- /regenerate creates another safe quote/style and increments poster_regenerate_count.
- /change quote shorter:
  shorten quote if possible.
- /change less cheesy:
  choose more direct systems/process language.
- /change more inspirational:
  choose more uplifting but still professional language.
- /change more minimal:
  reduce visual density and text.
- /change more cyberpunk:
  increase geometric/cyber visual accents, but keep readability.
- /change no subtitle:
  remove subtitle.
- /change <other free text>:
  record request and apply best-effort local adjustment.

Rendering requirements:
- Image size exactly 1080x1920.
- Background dark by default.
- Quote should be large, centered or slightly above center.
- Use safe margins of at least 100 px.
- Use high contrast text.
- Use theme text/accent colors if available.
- Use a monospaced or clean built-in font.
- Do not include the raw check-in.
- Do not include the private innovation takeaway directly unless it has been made public-safe.
- Do not include random extra generated words.
- Do not rely on AI rendering text.
- Use Pillow text rendering only.

Font behavior:
- Use common Windows fonts if available:
  Cascadia Mono
  Consolas
  Segoe UI
  Arial
- If font lookup fails, use Pillow default font.
- Do not include or distribute font files.
