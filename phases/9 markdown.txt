## Phase 9 Goal

Implement safe per-monitor wallpaper application on Windows.

Important: Phase 9 is the first phase that may actually change Windows wallpapers, but only with explicit confirmation.

Default behavior must be dry-run. Real application requires:

```text
/apply wallpapers --confirm
```

Dry-run commands:

```text
/apply wallpapers
/apply wallpapers --dry-run
```

Rollback command:

```text
/reset wallpapers
```

## Phase 9 Implementation Shape

Prefer adding these modules:

```text
src/chaseos/wallpaper/plan.py
src/chaseos/wallpaper/applier.py
src/chaseos/wallpaper/rollback.py
src/chaseos/wallpaper/windows_desktop_wallpaper.py
```

Suggested responsibilities:

- `plan.py`: build a `WallpaperApplyPlan` from latest wallpaper manifest, public poster, and monitor mapping.
- `windows_desktop_wallpaper.py`: thin Windows-only `IDesktopWallpaper` COM wrapper.
- `rollback.py`: save/load previous per-monitor wallpaper state.
- `applier.py`: dry-run, confirmed apply, reset/rollback orchestration.

If COM support needs a dependency, add it Windows-only:

```toml
comtypes; platform_system == "Windows"
```
