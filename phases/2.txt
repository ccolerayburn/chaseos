Phase 2: Build the terminal-style tray shell for ChaseOS.

Context:
Phase 1 created the ChaseOS scaffold at:

C:\_Codex\chaseos

The project uses a src-layout package and should run with:

python -m chaseos

Respect all rules in AGENTS.md.

Goal:
Build the first real ChaseOS shell: a Windows tray app that opens a small terminal-style dark command window. This phase is only the app shell and command input loop. Do not implement the full ritual state machine yet. Do not generate wallpapers yet. Do not apply wallpapers yet.

Product direction:
ChaseOS should feel like a quiet command console, not a dashboard.

UI requirements:
1. Use PySide6.
2. Add a system tray icon using QSystemTrayIcon.
3. Left-clicking or activating the tray icon should show/hide the main terminal window.
4. Right-click tray menu should include:
   - Start 15-Minute Sequence
   - Quick Theme Shift
   - Today’s Public Poster
   - Regenerate Wallpapers
   - Open Poster Archive
   - Settings
   - Reset Desktop
   - Exit

5. Main terminal window:
   - Default size: 720x520
   - Minimum size: 560x380
   - Resizable
   - Dark gray background
   - Muted darker yellow/amber monospaced text
   - Use Cascadia Mono if available, otherwise Consolas or another monospace font
   - Header line showing:
     CHASEOS :: START SEQUENCE
     status/timer placeholder
   - Scrollable terminal output area
   - Single command input line at bottom

6. Visual style:
   - No dashboard cards
   - No visual theme preview
   - No bright neon UI
   - No large panels
   - Text-first
   - Calm dark console aesthetic

7. Initial terminal output:
   When the window opens, show:

   chaseos> ready.
   chaseos> type /start to begin.
   chaseos> type /help for commands.

8. Input behavior:
   - Enter submits current input.
   - Escape minimizes/hides the window to tray.
   - Ctrl+L clears visible terminal output.
   - Submitted input should appear as:
     user> <input>
   - ChaseOS responses should appear as:
     chaseos> <response>
   - System/internal messages should appear as:
     system> <message>

Command handling:
Implement command parsing in:

src/chaseos/app/command_router.py

Support these commands in Phase 2:

- /help
- /start
- /clear
- /exit
- /reset
- /theme
- /poster
- /approve
- /regenerate
- /change <free text>
- /skip
- /status
- /monitors

Expected behavior for now:
- /help prints available commands.
- /start prints:
  chaseos> start sequence shell ready. ritual engine comes in phase 3.
- /clear clears visible terminal output.
- /exit exits the app cleanly.
- /reset prints a placeholder reset message.
- /theme prints a placeholder theme-plan message.
- /poster prints a placeholder public-poster message.
- /approve prints a placeholder approval message.
- /regenerate prints a placeholder regeneration message.
- /change <free text> echoes the requested change in a clean parsed form.
- /skip prints a placeholder skip message.
- /status prints current shell status.
- /monitors prints placeholder monitor roles:
  display 1 -> Public Signal Monitor
  display 4 -> Left Atmosphere Monitor
  display 2 -> Center Command Monitor
  display 3 -> Right Inspiration Monitor

Architecture requirements:
1. Keep GUI code separate from command parsing.
2. The command router should be testable without starting PySide6.
3. Use simple dataclasses or Pydantic models if helpful.
4. Keep terminal output formatting in one place where possible.
5. Avoid hardcoded Windows-only behavior except tray/window integration.
6. If tray icon image assets do not exist, create a simple fallback icon programmatically or use a safe default.
