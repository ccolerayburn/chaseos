The main Phase 10 risk is whether the monitor IDs stored by ChaseOS mapping are exactly the IDs needed by Windows IDesktopWallpaper.

Improve the Windows desktop wallpaper wrapper and/or plan builder so ChaseOS can reconcile monitors using the strongest available evidence.

Preferred matching order:
1. Existing saved IDesktopWallpaper monitor ID, if present.
2. Exact match against saved monitor device path, if present.
3. Rectangle match using monitor bounds from ChaseOS monitor detection and IDesktopWallpaper monitor rectangles.

Add support in the Windows desktop wallpaper wrapper for retrieving monitor rectangles if possible.

3. Add wallpaper diagnostics commands

Add these terminal commands:

/wallpaper status
/wallpaper diagnostics
/verify wallpapers

Command behavior:

/wallpaper status
Should report:
- Latest generated wallpaper manifest date/path.
- Latest public poster path, if found.
- Whether Display 1 is using public poster or fallback placeholder.
- Monitor mapping source.
- Whether rollback state exists.
- Whether last apply manifest exists.
- Whether Windows per-monitor wallpaper API is available.
- No wallpaper changes applied.

/wallpaper diagnostics
Should report:
- ChaseOS role mapping:
  - public
  - left
  - center
  - right
- Display aliases:
  - Display 1
  - Display 4
  - Display 2
  - Display 3
- Resolved IDesktopWallpaper monitor IDs, if available.
- Current wallpaper path per monitor, if available.
- Monitor rectangle/bounds, if available.
- Mapping confidence:
  - exact-id
  - exact-device-path
  - rectangle
  - unresolved
- Any warnings:
  - missing public poster
  - using Display 1 fallback placeholder
  - missing generated wallpaper
  - public image path appears to come from Lightroom/general photos
  - COM unavailable
  - role cannot be resolved to a Windows monitor
- No wallpaper changes applied.

Also save diagnostics JSON to:

%LOCALAPPDATA%\ChaseOS\wallpaper_state\last_wallpaper_diagnostics.json

/verify wallpapers
Should perform a strict preflight validation for real application:
- All four required roles exist:
  - public
  - left
  - center
  - right
- All four images exist.
- Public image is not from Lightroom/general photos.
- Public poster image is 1080x1920 when using the approved public poster.
- Private wallpapers are readable image files.
- Windows per-monitor wallpaper API is available on Windows.
- All four roles resolve to real IDesktopWallpaper monitor IDs.
- No duplicate monitor IDs.
- Rollback directory is writable.
- Apply manifest path is writable.
- No wallpaper changes applied.

Output should clearly end with either:

CHASEOS // WALLPAPER PREFLIGHT PASSED
No changes applied.
Run /apply wallpapers --confirm only if the mapping above looks correct.

or:

CHASEOS // WALLPAPER PREFLIGHT FAILED
No changes applied.
Fix the items above before using /apply wallpapers --confirm.
