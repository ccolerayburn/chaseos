Implement Phase 11: Headless command runner and non-mutating wallpaper smoke workflow.

Goal:
Add a safe command-line execution path so ChaseOS commands can be run from PowerShell without launching the GUI terminal. This is for verification, diagnostics, and repeatable smoke testing.

Do not redesign the app.
Do not duplicate command logic.
Use the existing command router wherever possible.
Do not apply wallpapers automatically.
Do not run /apply wallpapers --confirm during automated verification.
Do not run /reset wallpapers during automated verification.

Phase 11 requirements:

1. Preserve existing GUI behavior

Running this with no extra args must still launch the normal tray app:

.\.venv\Scripts\python.exe -m chaseos

Do not break the PySide6 tray shell.
Do not require CLI flags for normal GUI use.

2. Add single-command headless mode

Add support for:

.\.venv\Scripts\python.exe -m chaseos --command "/wallpaper status"

Also support a short alias if clean to implement:

.\.venv\Scripts\python.exe -m chaseos -c "/wallpaper status"

Behavior:
- Run the provided command through the same command router used by the terminal UI.
- Print the command output to stdout.
- Do not open the tray app.
- Do not open the terminal window.
- Return useful process exit codes.

Suggested exit codes:
- 0: command completed successfully.
- 1: command ran but reported a validation/preflight failure.
- 2: invalid CLI usage.
- 3: blocked because the command would mutate desktop state without explicit CLI unlock.

Use existing project conventions if there is already an exit-code pattern.

3. Add headless mutation safety gate

In headless mode, block desktop-mutating commands by default.

These commands must be blocked unless an explicit CLI unlock flag is provided:
- /apply wallpapers --confirm
- /reset wallpapers

Add an explicit unlock flag:

--allow-desktop-changes

Behavior:
- /apply wallpapers and /apply wallpapers --dry-run are allowed without --allow-desktop-changes because they are non-mutating.
- /wallpaper status is allowed.
- /wallpaper diagnostics is allowed.
- /verify wallpapers is allowed.
- /apply wallpapers --confirm is blocked unless --allow-desktop-changes is present.
- /reset wallpapers is blocked unless --allow-desktop-changes is present.
- Even when --allow-desktop-changes is present, /apply wallpapers should still require the command itself to include --confirm before real apply occurs.
- Never weaken the existing Phase 9/10 validation.

Blocked command output should be clear, for example:

CHASEOS // COMMAND BLOCKED

This command can change desktop wallpaper state.
Headless mode blocks it unless --allow-desktop-changes is provided.

No changes applied.

4. Add command script mode

Add support for running multiple commands from a text file:

.\.venv\Scripts\python.exe -m chaseos --script .\wallpaper_smoke.txt

Script behavior:
- One command per line.
- Blank lines ignored.
- Lines starting with # ignored.
- Commands are run in order.
- Output should clearly separate command sections.
- The process should continue through all non-mutating commands even if one command fails validation.
- Mutating commands should be blocked unless --allow-desktop-changes is provided.
- Return a non-zero exit code if any command fails or is blocked.
- Do not open GUI.

Example script:

# ChaseOS wallpaper smoke
/wallpaper status
/wallpaper diagnostics
/verify wallpapers
/apply wallpapers --dry-run

5. Add built-in wallpaper smoke mode

Add support for:

.\.venv\Scripts\python.exe -m chaseos --smoke wallpapers

This should run exactly these non-mutating commands in order:

/wallpaper status
/wallpaper diagnostics
/verify wallpapers
/apply wallpapers --dry-run

Behavior:
- Do not run /apply wallpapers --confirm.
- Do not run /reset wallpapers.
- Do not change wallpapers.
- Print all output to stdout.
- Save a text smoke report to:

%LOCALAPPDATA%\ChaseOS\wallpaper_state\last_wallpaper_smoke.txt

- Save a JSON smoke report to:

%LOCALAPPDATA%\ChaseOS\wallpaper_state\last_wallpaper_smoke.json

The JSON report should include:
- started_at
- completed_at
- commands
- per-command status
- per-command output
- aggregate_status
- exit_code

Suggested aggregate statuses:
- passed
- failed
- blocked

If /verify wallpapers fails because monitor mapping is unresolved, the smoke should still finish and write both smoke report files, then return a non-zero exit code.
