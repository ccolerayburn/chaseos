- "I feel depressed."

Output must reframe those into practical non-clinical signals.
For example:
- "I feel anxious" -> pressure: high, readiness: needs_calm
- "I feel depressed" -> mood_weight: heavy, readiness: needs_calm or needs_structure
- "ADHD brain is loud today" -> clarity: scattered, focus_friction: starting or prioritizing
Do not echo the clinical word back in the summary.

Startup mode selection rules:
1. If heavy mood and low energy and not urgent:
   Gentle Start.
2. If high pressure or overloaded and user asks calm:
   Calm Start.
3. If scattered/foggy/starting/prioritizing:
   Structured Start.
4. If high energy/restless and pressure is not high:
   Momentum Start.
5. If clear/focused/ready:
   Deep Work Start.
6. If tickets/incidents/queue/urgent appear:
   Triage Start unless pressure is high or overloaded, then Structured Start or Calm Start.
7. If headache/sensory load:
   prefer Calm Start or Structured Start with low intensity.
8. If uncertain:
   Structured Start.

Theme selection rules:
1. Calm Start:
   Prefer Synth Sanctuary or Arctic Interface.
   Intensity low.
   Density sparse or very_sparse.
   Motion none or minimal.

2. Structured Start:
   Prefer Obsidian Terminal or Chrome Monolith.
   Intensity low-medium.
   Density sparse.
   Motion none or minimal.

3. Gentle Start:
   Prefer Synth Sanctuary or Synthetic Sunrise.
   Intensity low.
   Density very_sparse or sparse.
   Motion none.

4. Momentum Start:
   Prefer Neon Noir or Violet Circuit.
   Intensity medium-high.
   Density medium.
   Motion minimal or normal.

5. Deep Work Start:
   Prefer Chrome Monolith or Obsidian Terminal.
   Intensity low-medium.
   Density very_sparse or sparse.
   Motion none.

6. Triage Start:
   Prefer Redline Protocol only when pressure is not high.
   If pressure is high, use Obsidian Terminal or Synth Sanctuary instead.
   Intensity medium unless pressure/body load requires lower.

7. Headache/sensory/body load:
   Lower intensity.
   Lower density.
   Avoid high red/orange.
   Avoid dense neon.

8. Low sleep/fatigue:
   Prefer softer contrast and lower density.
   Center display should be very minimal.

9. Creative/high-energy:
   Allow stronger cyberpunk intensity on Display 3.
   Keep Display 2 readable and clean.

10. Public Display 1:
   Always public innovation poster.
   Never use local general photos.
   Never use private check-in details.

Local photo usage rules for theme plan:
- Display 4 usually allows hybrid if no sensory/headache issue.
- Display 2 usually generated minimal.
- Display 3 can be generated or hybrid.

Rules:
- /change calmer:
  reduce intensity and density, prefer Synth Sanctuary or Arctic Interface.
- /change more cyberpunk:
  increase cyberpunk_intensity unless pressure/headache/sensory load is high.
- /change less visual noise:
  set density to very_sparse or sparse.
- /change more yellow:
  bias accent/text toward muted amber/gold.
- /change less cyan:
  reduce cyan to secondary detail only.
- /change use more local photos:
  set Display 4 and 3 to hybrid/local_photo unless body load suggests otherwise.
- /change no photos today or generated only:
  set Displays 4/2/3 to generated.
- /change darker:
  lower background/surface brightness.
- /change brighter:
  brighten surface slightly but keep dark-mode terminal/product aesthetic.
- /change more contrast:
  increase accent contrast.
- /change less contrast:
  soften accents.

Regenerate behavior:
- /regenerate should create a valid alternate theme plan using the same signals and startup mode.
- It should avoid returning the exact same family if reasonable.
- It should preserve safety constraints.
- It should increment regenerate_count in the session.

Integration with StartupSequence:
After user enters raw check-in:
1. LocalCheckInInterpreter runs.
2. Terminal prints practical interpretation summary.
3. ThemeGenerator creates ThemePlan.
4. Terminal prints THEME PLAN block.
5. Move to THEME_APPROVAL.
6. /theme prints current theme plan.
7. /change updates the current plan and records the change.
8. /regenerate regenerates the plan and records regenerate count.
9. /approve proceeds to MINDFULNESS.

- Poster plan can remain placeholder except it may reference the selected theme family/style.
