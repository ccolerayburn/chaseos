Implement Phase 13: Controlled live wallpaper apply acceptance test.

This phase is explicitly authorized to run the real wallpaper apply command once:

.\.venv\Scripts\python.exe -m chaseos --command "/apply wallpapers --confirm" --allow-desktop-changes

This command changes desktop wallpapers. Do not run it until after the preflight steps below pass.

Do not run /reset wallpapers automatically unless:
- the apply command fails after partial application,
- post-apply verification shows the wrong wallpaper path on any monitor,
- or an obvious safety/privacy violation is detected.

Do not make unnecessary code changes.
This phase is primarily an operational acceptance test.
Only patch code if the live apply exposes a bug.

Phase 13 goals:
1. Clean local smoke artifact.
2. Re-run non-mutating preflight.
3. Run one controlled live apply.
4. Verify applied wallpaper state through ChaseOS diagnostics and Windows wallpaper API.
5. Confirm rollback state and apply manifest were written.
6. Report results clearly.
7. Commit only documentation/test/code changes if any were required.

Step 3: Run non-mutating preflight

Run:

.\.venv\Scripts\python.exe -m chaseos --command "/doctor"
.\.venv\Scripts\python.exe -m chaseos --command "/assets status"
.\.venv\Scripts\python.exe -m chaseos --command "/wallpaper status"

.\.venv\Scripts\python.exe -m chaseos --command "/verify wallpapers"
.\.venv\Scripts\python.exe -m chaseos --command "/apply wallpapers --dry-run"

Requirements before live apply:
- /doctor must show core dependency readiness.
- comtypes must be available.
- Windows wallpaper API must be available.
- /assets status must show all expected images exist and are readable.
- /verify wallpapers must pass.
- /apply wallpapers --dry-run must resolve all four roles to real monitor IDs.
- No Display 1 candidate may come from Lightroom/general photos.
- No unresolved monitor IDs.
- No duplicate monitor IDs.

If any of these fail, stop. Do not run the live apply. Patch the underlying issue only if it is clearly a ChaseOS bug.

Step 4: Run the controlled live apply

Only after Step 3 passes, run:

.\.venv\Scripts\python.exe -m chaseos --command "/apply wallpapers --confirm" --allow-desktop-changes

Expected behavior:
- Rollback state is saved before any wallpaper is set.
- All four wallpapers are applied per monitor.
- No registry settings are modified.
- Explorer is not restarted.
- Admin rights are not required.
- Taskbar pins/icons are untouched.
- Public Display 1 receives the public poster only.
- Display 4 receives left atmosphere.
- Display 2 receives center command.
- Display 3 receives right inspiration.
