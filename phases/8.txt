src/chaseos/windows/display_detection.py
- Implement Windows monitor detection.
- Prefer a Windows-compatible method available without admin rights.
- Use stable identifiers where possible.
- Capture monitor rectangles and orientation.
- If true Windows display numbers are hard to obtain reliably, provide best available display labels and geometry.
- The code must fail gracefully on non-Windows or unsupported environments.

Possible implementation options:
- Use ctypes with EnumDisplayMonitors/GetMonitorInfo.
- Use pywin32 if already available or add it only if needed.
- Avoid requiring admin rights.
- Avoid changing system settings.

src/chaseos/windows/display_detection.py should expose:
- detect_monitors() -> list[DetectedMonitor]
- detect_monitor_layout() -> MonitorLayout or equivalent
- get_fallback_monitor_layout() -> MonitorLayout

Fallback behavior:
If detection fails, return a fallback layout matching the known setup:
- display 1: 1080x1920 portrait, role public_signal
- display 4: 1920x1080 landscape, role left_atmosphere
- display 2: 1920x1080 landscape, role center_command
- display 3: 1920x1080 landscape, role right_inspiration

The fallback should be clearly marked:
system> using fallback monitor layout. real detection unavailable.

  - auto_assign_known_layout(layout)
  - validate_assignments(layout)
  - summarize_monitor_layout(layout)

Auto-assignment:
When monitors are detected, attempt to auto-assign roles:
1. If a portrait monitor near the left side exists, assign it to public_signal.
2. For landscape monitors to the right:
   - leftmost landscape -> left_atmosphere
   - middle landscape -> center_command
   - rightmost landscape -> right_inspiration
3. If display labels 1, 4, 2, 3 are available:
   - display 1 -> public_signal
   - display 4 -> left_atmosphere
   - display 2 -> center_command
   - display 3 -> right_inspiration
4. If saved mappings exist and stable IDs still match, use saved mappings first.
5. If saved mappings are stale, warn and suggest reassignment.
